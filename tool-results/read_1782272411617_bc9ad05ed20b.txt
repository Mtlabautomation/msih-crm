  1311→  - **PDF export caps at 100 enquiries** (`enquiries.slice(0, 100)`) to keep the table within reasonable file size and rendering time. For very large datasets (>100), a future enhancement could add a "Export all" toggle or paginate the table across multiple PDF pages. Currently the autoTable will auto-paginate rows across PDF pages, but we slice to 100 to avoid excessive rendering.
  1312→  - **KPI cards on Audit Logs are based on current page only** (labeled "In current view" via hint). The API doesn't expose action totals across all pages; a future enhancement could add a `/api/audit-logs/stats` endpoint returning aggregated action counts. For now, the per-page counts + total counter at the bottom ("28 total events") give users a complete picture.
  1313→  - **Entity filter dropdown** is populated from the current page's logs only (via `useMemo` extracting distinct entities). If a user filters by an entity that's only present on page 1, switching to page 2 will empty the dropdown's source. In practice this is fine because the entity filter is a server-side filter (`entity=` query param) and the API returns the full filtered result set; the dropdown options just won't refresh when navigating pages. A future enhancement could pre-fetch all distinct entities via a separate endpoint.
  1314→  - **Date range filter on Reports is client-side** — it filters the 200 enquiries already loaded by the `/api/enquiries?limit=200` query. For datasets >200 enquiries, results older than the 200 most-recent will be missing from filtered output. A future enhancement could pass `dateFrom`/`dateTo` to the `/api/enquiries` API (which already supports these params per the route) — but that requires invalidating the queryKey and refetching, which is a more involved change. For now, 200 records is enough for demo/typical use.
  1315→  - **PDF font**: jspdf's default Helvetica doesn't render ₹ (INR symbol) — it'll appear as "Rs" or a placeholder in the PDF. The fmtINR helper uses Intl.NumberFormat which produces "₹17,10,000"; this string gets passed to autoTable. A future enhancement could either strip the ₹ and prefix "Rs.", embed a Unicode TTF font (jspdf supports `.addFont()`), or use a custom font face for proper ₹ rendering.

For datasets >100 tasks (the API default limit), some tasks may be missing from filtered results. The API supports `?limit=` up to 500, so increasing the limit would mitigate this. This is a deliberate UX choice (focus on one status at a time) but differs from typical "filter" semantics. If users find this confusing, a future enhancement could add a separate "Focus column" dropdown distinct from the status filter that gets saved with the view.

This is a minor inconvenience but keeps the shared component generic. A future enhancement could pass the `chips` array (like enquiries does) for a richer filter summary, but that would require each view to expose a chip-building function.

However, if the user had manually cleared filters and was viewing "All", navigating away and back will re-apply the default — which may or may not be desired. A future enhancement could persist `defaultApplied` in sessionStorage to prevent re-applying within a session.
